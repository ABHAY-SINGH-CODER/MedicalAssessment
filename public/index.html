API Reference
Three serverless endpoints — ready to integrate into any app.
POST /api/assess
Run a full medical risk assessment. Accepts patient info, symptoms, and optionally a base64-encoded medical image. Automatically selects MedGemma-4B (multimodal) when image is provided.
// Example request fetch('/api/assess', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ age: 45, sex: 'Male', symptoms: 'Chest pain, Shortness of breath', duration: '1–3 days', notes: 'Sharp pain radiating to left arm', image: '<base64 string>', // optional imageType: 'xray' }) })
GET /api/health
Health check endpoint. Returns API status, configured model names, available endpoints, and whether the HF_API_KEY environment variable is set.
// Example response { "status": "ok", "service": "MedGemma Risk Assessment API", "apiKeyConfigured": true, "models": { "textOnly": "google/medgemma-2b-it", "multimodal": "google/medgemma-4b-it" } }
GET /api/models
Lists available MedGemma models with capabilities, context window size, supported image types, and auto-selection logic.
// Example response { "models": [ { "id": "google/medgemma-2b-it", "type": "text-only", "free": true }, { "id": "google/medgemma-4b-it", "type": "vision + text", "free": true } ] }
